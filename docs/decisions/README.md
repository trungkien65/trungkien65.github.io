# Architecture Decision Records (ADR)

Directory for storing important architecture decisions.

## Format

Each ADR should have:

- **Title**: Short description
- **Context**: Problem to solve
- **Decision**: Chosen decision
- **Consequences**: Benefits / trade-offs

## Example

```
# ADR-001: Use Jotai instead of Redux

- Context: Need state management for the game.
- Decision: Jotai – compact atoms, no boilerplate.
- Consequences: Easy to learn, small bundle; no DevTools as powerful as Redux.
```
